The problem we’re solving


Sleep & circadian research depends on validated instruments, careful data collection, and reproducible pipelines.

Yet the tools connecting these layers are often:

  • fragmented across incompatible formats and platforms, paper-based methods
  • brittle — dependent on locally-installed, unversioned software
  • locked behind proprietary or expensive licences

We are building a stack of interconnected open-source tools to address this!

The Circadia Lab toolchain

Two apps for data collection → two R packages for analysis → one reproducible environment.


🌙 Sleep Diaries — participant-facing sleep diary app (iOS, Android, web);

📋 ScoreMe — lab/clinic questionnaire scorer for research cohorts;

💤 slumbR — R package: import & analyse Sleep Diaries exports;

🧮 tallieR — R package: import & analyse ScoreMe exports;

🐳 Circadia Base Docker — reproducible Python 3.11 + R 4.4 research environment.


All tools: open source · MIT licence (or similar)

Sleep Diaries

A consensus-based sleep diary app for iOS, Android and web.

  • Morning (13-item) and evening (5-item) diary entries
  • Validated questionnaires: ESS, ISI, MEQ, PSQI, DBAS-16, MCTQ, RU-SATED, STOP-BANG
  • Automatic sleep metrics after 14 entries — TST, SE, SOL, WASO
  • Research codes for structured studies · CSV + JSON export/import
  • Full Brazilian Portuguese localisation · PWA — installable, works offline

sleepdiaries.circadia-lab.uk

ScoreMe

A lab and clinic app for administering validated questionnaires to research cohorts.

  • 27 instruments across 5 domains: sleep, mental health, wellbeing, physical activity, neurodevelopmental
  • Rich participant profiles: code, demographics, group, site, session
  • Score history — re-administrations appended, nothing overwritten
  • Analytics tab: box plots, descriptive stats, completion rates, group comparisons
  • Custom questionnaire import via JSON spec · Brazilian Portuguese localisation

v1.0 · scoreme.circadia-lab.uk

slumbR

R companion for Sleep Diaries exports.

  • read_export() / read_study() — parse JSON exports
  • compute_sleep_vars() — derives TIB, TST, SE, SOL, WASO
  • diary_wide() — one row per participant × night with m_ / e_ pre.
  • score_questionnaire() — re-scores all 8 instruments; match the app

Supports: ESS, ISI, DBAS-16, MEQ, PSQI, RU-SATED, STOP-BANG, MCTQ


remotes::install_github("circadia-bio/slumbR")

tallieR

R companion for ScoreMe exports.

  • read_scoreme() / read_scoreme_dir() — parse exports; re-scores
  • scores_wide() / scores_long() — per participant, or full history
  • cronbach_alpha() / omega_reliability() — with exact CIs
  • interpret_all() — clinical interpretations for every result
  • load_instrument() — score custom instruments from JSON
remotes::install_github("circadia-bio/tallieR")

Circadia Base Docker

A reproducible computing environment for chronobiology and actigraphy research.


JupyterLab localhost:8889
Python 3.11: pyActigraphy, MNE, YASA, EntropyHub, scikit-learn, scipy, statsmodels

RStudio localhost:8787
R 4.4: GGIR, ActCR, circacompare, cosinor, lme4, brms, rstanarm

Fully pinned dependencies · shared volume · Apple Silicon native


git clone https://github.com/circadia-bio/circadiaBase_Docker && docker compose up

Design principles


🔓 Open source, MIT licensed — everything on github.com/circadia-bio;

🔀 Interoperable — apps export standard JSON; R packages read those exact files;

🔂 Reproducible — pinned dependencies in Docker; versioned releases with DOIs;

🔬 Research-grade — algorithm parity between app scoring and R re-scoring;

📦 Accessible — web PWAs (no installation); R packages install in one line;
